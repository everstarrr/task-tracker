Отчет о разработке проекта

Дата создания: 19.12.2024
Название проекта: Трекер задач - веб-приложение для управления и отслеживания выполнения задач


Основные задачи проекта

Разработать полнофункциональное веб-приложение с использованием современного стека технологий:
- Применить ключевые концепции Vue 3: синхронизацию данных, обработку событий, вычисляемые свойства и наблюдатели
- Реализовать работу с пользовательскими формами с применением двусторонней привязки и валидацией
- Внедрить условное отображение контента в зависимости от состояния приложения
- Организовать обработку коллекций данных с функциями фильтрации и переупорядочения
- Разработать модульную архитектуру на основе компонентов с передачей параметров и обменом сигналами
- Настроить навигацию между страницами приложения с помощью маршрутизации
- Освоить работу с жизненным циклом компонентов и управлением ссылками на элементы

Создать серверное приложение на Python для обработки запросов и управления данными.
Реализовать развертывание приложения в контейнерах используя Docker.
Документировать архитектуру и ключевые решения проекта.


Компоненты и их функциональность

Архитектура фронтенда

В проекте реализованы следующие компоненты:

- **TaskList.vue** — основной компонент для представления полного каталога задач, включает в себя логику фильтрации по различным критериям и сортировку по временным меткам или названиям
- **TaskItem.vue** — самостоятельная единица отображения, отвечающая за визуализацию отдельной записи, использует входные параметры для получения информации и механизм событий для коммуникации с родительским уровнем
- **TaskForm.vue** — интерактивная форма для введения новых записей и внесения изменений в существующие с встроенной проверкой корректности данных
- **AppHeader.vue** — верхняя часть интерфейса с элементами навигации и логотипом
- **AppFooter.vue** — нижняя часть интерфейса с дополнительной информацией и ссылками
- **LayoutCard.vue** — многоцелевой контейнер для отображения информации, демонстрирует использование механизма передачи шаблонов между компонентами

Вычисляемые свойства и отслеживание изменений

В компоненте TaskList функциональность обеспечивается вычисляемыми свойствами:

- **filteredTasks** — фильтрует задачи согласно их статусу (все задачи, завершённые, активные)
- **sortedTasks** — переупорядочивает список по времени создания или в алфавитном порядке

Для отслеживания изменений используется механизм наблюдения (watch):
- В TaskList отслеживаются изменения параметров фильтра и вывод соответствующей информации в консоль
- В TaskForm контролируются изменения во входных полях для сброса сообщений об ошибках при валидации

Система слотов и передачи данных

В компоненте LayoutCard реализована гибкая система передачи шаблонов с тремя вариантами:

- Стандартный механизм для размещения основного наполнения
- Именованные области для заголовка и заключительной части  
- Специализированный тип с ограниченной областью видимости (actions), который предоставляет данные обратно в вызывающий компонент, такие как флаг активного состояния и переключатель статуса

Система маршрутизации

Навигационная структура приложения построена с использованием Vue Router и включает следующие маршруты:

- `/` — основная страница приложения
- `/tasks` — полный перечень всех доступных задач
- `/tasks/new` — интерфейс для добавления новой записи
- `/tasks/:id/edit` — страница редактирования с динамическим параметром идентификатора задачи
- `/:pathMatch(.*)* ` — страница сообщения об ошибке 404 при обращении к несуществующему адресу

Все маршруты содержат именованные идентификаторы, что дает возможность выполнять переход между страницами программно, используя конструкцию типа this.$router.push({ name: 'tasks' }).

Серверная реализация и интеграция

Бэкенд разработан на языке Python и реализует:

- Набор операций для работы с задачами (создание, чтение, изменение, удаление)
- Сохранение информации в структурированном текстовом формате (JSON)
- Использование типизированных классов для проверки входящих данных
- Промежуточное программное обеспечение для разрешения перекрёстных запросов между доменами (CORS)

Ключевые фрагменты кода

Итерирование по коллекции с передачей информации между компонентами:

```vue
<TaskItem
  v-for="task in sortedTasks"
  :key="task.id"
  :task="task"
  @delete="$emit('delete', $event)"
  @toggle="$emit('toggle', $event)"
  @edit="$emit('edit', $event)"
/>
```

Двусторонняя привязка данных с модификаторами в форме:

```vue
<input
  id="title"
  type="text"
  v-model.trim="form.title"
  placeholder="Введите заголовок задачи"
/>

<textarea
  id="description"
  v-model.lazy="form.description"
  placeholder="Описание задачи"
></textarea>
```

Использование специализированного слота с передачей данных:

```vue
<template #actions="{ toggleHighlight, isHighlighted }">
  <button @click="goToTasks">Перейти к задачам</button>
  <button @click="toggleHighlight">
    {{ isHighlighted ? 'Убрать выделение' : 'Выделить карточку' }}
  </button>
</template>
```

Инструкция по развертыванию в контейнерах

Для запуска приложения в изолированной окружении выполните следующие команды в терминале:

```bash
docker compose build
docker compose up
```

После завершения процесса приложение станет доступно по адресу http://localhost:8080

**Описание конфигурации:**
- Интерфейс собирается с помощью инструмента Vite и раздаётся через веб-сервер Nginx
- Серверная часть запускается с использованием Uvicorn на внутреннем порту 8000


Заключение

В ходе разработки проекта получен практический опыт работы с ключевыми концепциями современных веб-приложений. Освоены методы организации компонентной архитектуры на основе Vue 3, включая взаимодействие между элементами через механизмы передачи параметров и событий, реализация гибких интерфейсов с помощью шаблонов, а также управление состоянием через вычисляемые свойства и наблюдатели. 

Практическое применение маршрутизации обеспечило углубленное понимание принципов навигации в одностраничных приложениях. Серверная реализация на Python продемонстрировала базовые подходы к разработке API и управлению данными. Использование Docker позволило познакомиться с процессом контейнеризации и упрощением развертывания приложений в различных окружениях.
